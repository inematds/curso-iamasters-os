🧠 O que é o iAmasters OS

Conceito, arquitetura em 3 camadas, e o problema que o OS resolve face ao Claude Code vanilla.

O que é:

O Claude Code "vanilla" é poderoso mas amnésico — cada sessão começa de zero. Tens de explicar quem és, que stack usas, que cliente é este, qual a tua voz. Resultado: tempo perdido e outputs genéricos.

Por que aprender:

Porque sem entender o problema não vês o valor do OS. O iAmasters OS existe para resolver exatamente isto: memória persistente entre sessões.

Conceitos-chave:

Sessão amnésica · contexto perdido · onboarding repetido · outputs genéricos · custo de re-explicação.

O que é:

O OS empilha 3 camadas: Sinapsis (engine de memória, vendorizado do Luis Pitik) · Camada OS (brand-context, agent-context, projetos, multi-cliente) · 23 Skills curadas (marketing, automation, tools, etc.).

Por que aprender:

Saber onde cada coisa vive permite-te debugar quando algo falha. Sinapsis em ~/.claude/, OS no repo. Não os confundas.

Conceitos-chave:

Vendoring · engine vs camada · skills on-demand · operator-state global vs context local.

O que é:

A árvore canónica: .claude/skills/ (23 skills + commands) · brand-context/ (voz, ICP) · context/ (me.md, work.md, etc.) · clients/ (multi-cliente) · vendor/sinapsis/ (engine).

Por que aprender:

Para saberes que ficheiros editar, quais não tocar, e o que entra no git público vs privado.

Conceitos-chave:

Separação user-data vs system-files · .gitignore extensivo · soul.md (estático) vs me.md (dinâmico).

O que é:

Cada SKILL.md tem um description: no frontmatter. O Claude lê todas no início da sessão e ativa a relevante consoante o que escreves. Tu não chamas — ela encontra-te.

Por que aprender:

Para escreveres descriptions que ativem corretamente. E para refinares prompts quando a skill errada disparar.

Conceitos-chave:

Progressive disclosure · description como trigger · 50-500 chars · verbos de intenção.

O que é:

Operadores IA freelance · agências pequenas (2-10) · formadores/educadores · consultores/empreendedores. O OS adapta-se a cada um via templates verticais.

Por que aprender:

Para identificares onde te encaixas e que template usar quando criares clientes (Trilha 3).

Conceitos-chave:

Avatar · vertical · template · personalização vs convenção.

O que é:

OS = grátis (MIT). Mas requer Anthropic Pro ($20/mês mínimo) ou Max ($100-200/mês). Prerequisitos técnicos: Node.js 18+, Python 3.9+, Git, Claude Code CLI.

Por que aprender:

Para não te assustares quando vires a fatura. Plano Free não chega — os modelos bons não estão lá.

Conceitos-chave:

Pro vs Max · Firecrawl API (opcional) · membership iAmasters Academy (separada da Anthropic).

⚙️ Instalação e Install Gate

State machine persistente, validação profunda, e como o gate impede "instalações fantasma".

O que é:

Lista do que tem de estar instalado antes de correres o `install.sh`. O script valida tudo e bloqueia se algo crítico falta.

Por que aprender:

Para resolveres falhas de instalação. Windows + Microsoft Store Python é o erro mais comum.

Conceitos-chave:

Deteção Python multi-plataforma · Node ≥18 · OS detection (macOS/Linux/Windows-bash).

O que é:

Prereqs → Sinapsis → Capa OS → soul.md/decisions-log.md → install-gate.sh → settings.json registration. ~30 segundos.

Por que aprender:

Para correres `--resume` quando algo trava, ou `--force-reinstall` para reiniciar limpo sem perder dados.

Conceitos-chave:

Idempotência · output estruturado [OK]/[SKIP]/[WARN]/[ERROR] · backup automático antes de force.

O que é:

JSON com 6 fases: prereqs, sinapsis-engine, context-files, operator-state, welcome-completed, deep-dive-completed. Cada uma tem `status` (done/failed/pending/in-progress/skipped).

Por que aprender:

Para perceber por que o agente diz "instala primeiro" quando achas que já instalaste. O state file não mente.

Conceitos-chave:

Validação profunda · drift detection · 5 fases required + 1 deferrable.

O que é:

Hook bash + node em `~/.claude/skills/_install-gate.sh`. Lê state, injeta `additionalContext` ao modelo se há fases pendentes. O modelo recebe instrução clara antes de ver a tua mensagem.

Por que aprender:

É a peça que evita "o Claude inventou que instalou". Enforcement estrutural, não confia no modelo.

Conceitos-chave:

SessionStart hook · additionalContext JSON · enforcement vs confiança · fallback se node não existe.

O que é:

`/install` lê state, identifica próxima fase, executa (ou diz-te para correres bash no terminal). `/install-status` é read-only — mostra o dashboard sem tocar em nada.

Por que aprender:

Para destrancar instalações a meio. `/install --resume` retoma onde ficou, `--force-reinstall` reinicia tudo (com confirmação).

Conceitos-chave:

Reentrante · fases técnicas (terminal) vs conversacionais (Claude Code).

O que é:

A skill `health-check` (invocada por `/doctor`) faz validação profunda: JSON parseable, hooks executáveis, conteúdo >100 chars em ficheiros críticos. Deteta drift entre state e disco.

Por que aprender:

Quando algo falha sem dares conta — apagaste um ficheiro, mudaste config — `/doctor` diz-te o que está partido e propõe fixes.

Conceitos-chave:

🟢🟡🔴 report · auto-fix com confirmação · cruzamento com state machine.

🎨 Brand Context

Voice profile com 3 registos A/B/C, ICP, positioning. A peça que faz os outputs soarem a TI, não a IA genérica.

O que é:

4 subpastas: voice/ (profile + 3 registos + samples) · positioning/ · icp/ · assets/ (logos, cores). Privada (.gitignore).

Por que aprender:

Para saberes que ficheiros editar. Todas as skills marketing-* leem aqui antes de gerar output.

Conceitos-chave:

Camada estática · privada por defeito · 1 brand-context por repo/cliente.

O que é:

A formal (email premium, propostas) · B divulgativo (LinkedIn, blog, 95% do conteúdo público) · C próximo (WhatsApp, DM, comentários). Cada um com vocabulário próprio.

Por que aprender:

Porque escreves muito diferente em email premium vs WhatsApp. 1 voice genérico produz outputs homogéneos.

Conceitos-chave:

Spectrum 0-10 em 5 dimensões (formality, directness, humor, authority, warmth) · auto-deteção de registo por plataforma.

O que é:

Perfil cliente ideal: dor exata · linguagem que usa · buying triggers · anti-ICP. Skill `marketing-icp` constrói via análise de clientes bons vs maus.

Por que aprender:

Para a copy falar a língua deles, não a tua. ICP plano = copy plana.

Conceitos-chave:

Customer journey · frases de validação · anti-ICP (clientes que não queres).

O que é:

Porque te escolhem a ti e não a outro. Skill `marketing-positioning` analisa concorrentes, identifica gaps, propõe 3-5 ângulos.

Por que aprender:

Para ter direção clara nos outputs. Sem positioning, a copy é "consultor genérico", não "TU".

Conceitos-chave:

Vertical-specialist · use-case-specialist · método próprio · anti-vendehumos.

O que é:

A skill `marketing-brand-voice` v2 tem dupla rota: artefactos reais (URLs públicos via Firecrawl) ou 15 simulações reais (5 por registo). Gera 8 ficheiros em voice/.

Por que aprender:

Para configurares o teu voice em 15-20 min sem ter de redigir tudo à mão.

Conceitos-chave:

Firecrawl API (opcional) · validação intermédia · audit-prompt.md reutilizável.

O que é:

Editar `voice-profile.md` diretamente · re-correr skill com `refine: true` · trocar positioning quando muda ângulo · refrescar ICP após cada batch de clientes novos.

Por que aprender:

Brand context evolui. Reservar 30 min por trimestre para refinar paga-se 10x.

Conceitos-chave:

Refine vs rebuild · samples como ground-truth · vocabulary.md (palavras-assinatura).
