1.1 ~50 min

🧠 O que é o iAmasters OS

Conceito, arquitetura em 3 camadas, estrutura do repo, como skills se ativam.

O que é:

Claude Code vanilla começa cada sessão de zero. Não sabe quem és, que stack, que cliente, qual a tua voz. Tens de re-explicar tudo.

Por que aprender:

Sem entender o problema, não vês o valor do OS. Memória persistente é a peça que muda tudo.

Conceitos-chave:

Sessão amnésica · contexto perdido · re-onboarding · outputs genéricos · custo invisível.

O que é:

Sinapsis (engine de memória, vendorizado) · Camada OS (brand-context, agent-context, projetos) · 23 Skills curadas (marketing, automation, tools).

Por que aprender:

Saber onde cada coisa vive permite debugar. Sinapsis em ~/.claude/, OS no repo.

Conceitos-chave:

Vendoring · engine vs camada · skills on-demand · operator-state global vs context local.

O que é:

.claude/skills/ · brand-context/ (privado) · context/ · clients/ (privado) · vendor/sinapsis/ (não tocar).

Por que aprender:

Para saberes que ficheiros editar, o que não tocar, e o que vai para git público vs privado.

Conceitos-chave:

Separação user-data vs system-files · .gitignore extensivo · soul.md (estático) vs me.md (dinâmico).

O que é:

Cada SKILL.md tem um description: no frontmatter. Claude lê todas no início e ativa a relevante consoante o que escreves.

Por que aprender:

Para escreveres descriptions que ativem corretamente. E refinares prompts quando a skill errada disparar.

Conceitos-chave:

Progressive disclosure · description como trigger · 50-500 chars · verbos de intenção.

O que é:

Sinapsis vive em vendor/sinapsis/ como espelho intacto do upstream (Luis Pitik). Qualquer modificação local perde-se em cada update.sh.

Por que aprender:

Para não perderes tempo editando ficheiros que serão sobrescritos. Mudanças vão para fork ou issue upstream.

Conceitos-chave:

Vendor lock-free · upstream-friendly · forking strategy · separação engine/wrapper.

O que é:

O OS não é "produto fechado". É um substrato. Skills novas chegam via /install-skill <url>, MCPs via /install-mcp. Quem partilha, multiplica.

Por que aprender:

Para participares ativamente em vez de consumires passivamente. Tu também publicas.

Conceitos-chave:

Catálogo curado · validação antes de globalizar · cultura de fork · contribuição open source.

👥 Para quem é + custo real

4 avatares, custo Anthropic (Pro/Max), prerequisitos técnicos para correr o OS.

O que é:

Operas sozinho ou em equipa de 1-2. Implementas IA nos processos dos clientes. Template: freelance-ia.

Por que aprender:

Para identificares se este és tu. O brand-context e a estrutura multi-cliente foram pensadas para este perfil primeiro.

Conceitos-chave:

Implementação + manutenção · ticket project-based · 1-2 pessoas · brand pessoal forte.

O que é:

5-30 clientes em simultâneo, entregáveis mensais (content calendar, ads, reports). Cada cliente com a SUA voice. Template: agencia-marketing.

Por que aprender:

Para perceberes como a estrutura clients/ escala. Voice isolada por pasta evita contaminação entre clientes.

Conceitos-chave:

MRR · multi-tenant · isolamento de voice · workflows repetíveis · capacidade vs personalização.

O que é:

A tua marca pessoal É o produto. Audiência nas redes (LinkedIn, X, YouTube). Alta frequência de output. Template: formador-online.

Por que aprender:

Para perceberes como skills marketing-* + repurposing aceleram produção sem perder voz.

Conceitos-chave:

Brand pessoal · content repurposing · cursos/comunidade · cadência alta.

O que é:

Engagements longos (3-12 meses) com empresas médias/grandes. Decks formais, frameworks, voz autoridade. Template: consultoria-b2b.

Por que aprender:

Para configurares voice profile no registo A (formal) e usares positioning com peso técnico.

Conceitos-chave:

High-ticket · engagement longo · entregáveis formais · authority positioning.

O que é:

OS = grátis (MIT). Anthropic Pro = $20/mês (mínimo real). Max = $100-200/mês (uso intensivo). Firecrawl opcional.

Por que aprender:

Para não te assustares com a fatura. Free não dá acesso aos modelos bons (Opus, Sonnet 4+).

Conceitos-chave:

Pro vs Max · uso vs subscription · API opcional · membership iAmasters Academy separada.

O que é:

Claude Code CLI + Node.js ≥18 + Python ≥3.9 (NÃO Microsoft Store em Windows) + Git. macOS, Linux, Windows (Git Bash/WSL).

Por que aprender:

Para evitares o erro #1 (Python da Microsoft Store em Windows). O installer valida tudo antes de correr.

Conceitos-chave:

Deteção multi-plataforma · validação prereqs · WSL como alternativa Windows · versões mínimas.
