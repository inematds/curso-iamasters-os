🪄 Meta skills

As skills que configuram, mantêm e estruturam o teu pensamento. Os 6 pilares que fazem o OS conhecer-te.

O que é:

Entrevista conversacional adaptativa em 4 sub-fases (W1 identidade · W2 negócio · W3 foco · W4 config). Cada uma escreve ficheiro e atualiza state antes de avançar.

Por que aprender:

É a porta de entrada do OS. Se abandonas a meio, retoma exatamente onde ficaste.

Conceitos-chave:

8 dimensões críticas · 10-12 min · 100 chars mínimo por ficheiro.

O que é:

4 blocos: Pessoa (7) · Negócio (6) · Equipa (6, gate condicional) · Foco e visão (5-7). Branching adaptativo.

Por que aprender:

Diferença entre outputs "decentes" e outputs que parecem mesmo teus. Investe-se 15-20 min para ganhar meses.

Conceitos-chave:

Tom de entrevistador profissional · sem validação coach · escuta, anota, próximo.

O que é:

4 passos: detetar estado, carregar continuidade (daily summary), sincronizar skills, cumprimento contextual.

Por que aprender:

Nunca começas do zero. "Olá!" é mau cumprimento; "Ontem fechaste positioning v2..." é bom.

Conceitos-chave:

suggestDeepDive flag · continuidade · curto sem fluff.

O que é:

5 passos: recap, sync skills, update CLAUDE.md registry, append learnings, daily summary + commit.

Por que aprender:

Sem fecho, o início de amanhã não tem o que ler. 2 minutos no fim poupam 15 no início.

Conceitos-chave:

Daily summary formato Did/Pending/Tomorrow · não corre sozinho · commit conventional.

O que é:

Ficheiro append-only com formato canónico: decisão, raciocínio, contexto, quando reconsiderar.

Por que aprender:

Evita o agente contradizer-se (e tu também) ao longo do tempo. Padrão de Luis Pitik.

Conceitos-chave:

Critério "quando reconsiderar" mensurável · invocado por outras skills (six-hats, positioning).

O que é:

welcome: primeiro deliverable em 5 min (3 hooks + plano). six-hats: análise multi-perspectiva para decisões grandes.

Por que aprender:

São as skills que entregam "wow" — adoção e decisões.

Conceitos-chave:

Quick-win usa Firecrawl + visual-explainer · six-hats modo rápido/profundo · encadeia decisions-log.

✍️ Marketing skills

Copy com voice carregada, repurposing 1→N, email sequences, padrão de skill collaboration.

O que é:

Output mais frequente do OS. Plataforma + brand voice + register A/B/C auto-detetado. 1-3 variações + verifier obrigatório.

Por que aprender:

É a skill estrela do dia-a-dia. Saber inputs e overrides poupa horas de prompt-engineering.

Conceitos-chave:

Plataforma muda tudo · purpose muda tom · pedir 2-3 variações.

O que é:

Converte fonte (YouTube, blog, transcript, reunião) em pacote multiplataforma (LinkedIn, X thread, newsletter, IG, clips).

Por que aprender:

Multiplica o ROI de cada peça grande que produzes. 15-20 min de trabalho teu por talk.

Conceitos-chave:

Invoca copywriting internamente · cada peça passa pelo gate individual · áudio local não suportado v0.2.

O que é:

Sequências multi-email: welcome, nurture, re-engagement, post-purchase, event-based, educational drip.

Por que aprender:

Para fluxos automatizados (lista própria opt-in). NÃO confundir com cold outreach B2B.

Conceitos-chave:

Cadências por tipo · métricas-alvo · 6 perguntas iniciais (tipo, audiência, objetivo, tom, ferramenta, restrições).

O que é:

Skills que geram brand-context (vivem em brand-context/positioning/ e icp/), consumidas pelas outras marketing-*.

Por que aprender:

Para entenderes a dependência: sem brand-context bem feito, copywriting/email/repurposing produzem outputs genéricos.

Conceitos-chave:

ICP refrescar após cada batch · positioning só com 3 gatilhos legítimos. Ver T3 para deep dive.

O que é:

Pipeline canónica para qualquer output marketing. JSON passado entre skills com estado completo.

Por que aprender:

Para debugar quando skill "falha" — normalmente é problema de composição, não da skill individual.

Conceitos-chave:

Score >=8 entrega · <8 vai a humanizer · falhas típicas (voice não carregada, platform errada).

O que é:

Skills marketing custam tokens, tempo e ruído. Override manual com "escreve sem skill, é interno".

Por que aprender:

Saber NÃO ativar é tão importante quanto saber ativar. Brainstorm não precisa de verifier.

Conceitos-chave:

Regra: >1 leitor externo, ativa. Se é só para ti, não. 3 frases de override que respeita.

🛠️ Tools + Visualização

Humanizer, verifier, firecrawl, visual-explainer. Threshold por plataforma e patterns de skill collaboration entre tools.

O que é:

Deteta 50+ AI-tell patterns: vocabulário-sinal, frases-template, estrutura simétrica, hedging excessivo, listas tripletas, transições mecânicas.

Por que aprender:

É a defesa contra outputs "suspeitosamente IA". Modo standalone ou dentro de pipeline.

Conceitos-chave:

Score 0-10 (10 = humano puro) · reescrita comentada · invocada pelo verifier.

O que é:

Gate que corre 4 checks: humanizer (30%) · voice match (30%) · length per platform (20%) · factuality flags (20%).

Por que aprender:

Para entender por que outputs ficam bloqueados ou marcados com warnings.

Conceitos-chave:

Pass / pass-with-warnings / fail · JSON estruturado com scores e suggestions.

O que é:

Wrapper de Firecrawl API para scraping sem bot blockers. Se faltar API key, cai para WebFetch nativo e avisa.

Por que aprender:

É a fonte de dados frescos do OS. 500 créditos grátis em firecrawl.dev.

Conceitos-chave:

3 passos: verificar key · validar input · extrair markdown. Cache 24h.

O que é:

Gera HTML5 autocontido com CSS+JS inline. Mobile-first. Pequeno o suficiente para DM, WhatsApp, email attach.

Por que aprender:

Para análises partilháveis (positioning, six-hats, quick-win). NÃO usar para outputs internos <200 palavras.

Conceitos-chave:

Dark mode default · acento iAmasters · Inter via Google Fonts · output em projects/visual/.

O que é:

Threshold de humanizer varia por canal. Email cold ≥8, LinkedIn ≥7, WhatsApp ≥6, notas internas N/A.

Por que aprender:

Para evitar gates desnecessários. Threshold uniforme bloqueia onde não importa e deixa passar onde importa.

Conceitos-chave:

Ajusta ao teu ICP · documenta em audit-prompt.md · +1 ICP técnico, -1 ICP executivo.

O que é:

4 patterns: gate composto (verifier compõe humanizer) · source→skill→visual · fallback graceful · loop com sanity break.

Por que aprender:

Reconhecer patterns acelera debug e permite construires as tuas próprias skills compostas.

Conceitos-chave:

Debug por JSON · 3 iterações max no loop · skill = composer de outras.
