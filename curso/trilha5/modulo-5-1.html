🤔 Quando usar (e quando não) multi-cliente

Multi-cliente não é "obrigatório". É uma capacidade opcional do OS. Ativá-lo sem o precisar adiciona fricção; não o ativar quando devias leva a copy errada, mistura de contextos e leaks de info entre clientes.

💡 Regra de ouro

Se trabalhas com 2+ clientes que têm voice, ICP e positioning distintos → multi-cliente. Caso contrário, fica na raiz.

O setup multi-cliente acrescenta ~30s por sessão (cd, claude, wrap-up). Compensa só quando há separação real para manter.

✓ Ativa multi-cliente se

  • ✓És freelance / agência com 2+ clientes em simultâneo
  • ✓Cada cliente tem brand voice distinta da tua
  • ✓Queres separação clara de outputs no filesystem
  • ✓Geras deliverables que não devem ter pegada tua
  • ✓Queres decisions-log independente por cliente

✗ NÃO ativar se

  • ✗És single-business (uma marca/produto só)
  • ✗Os "clientes" são verticais do mesmo produto
  • ✗Trabalhas só na tua marca pessoal
  • ✗Clientes querem que uses a TUA voice (consultor cara visível)
  • ✗Estás a começar e tens 1 cliente piloto

👤 Template freelance-ia

O vertical mais comum entre operadores iAmasters. Para quem opera sozinho ou em equipa de 1-2, com 3-10 clientes em paralelo, ticket médio 5-50K€ por projeto.

🎯 Voice profile pré-configurada

  • •Formality: 6/10 — profissional mas acessível
  • •Directness: 8/10 — vai ao ponto, sem rodeios
  • •Warmth: 7/10 — humano, não corporativo
  • •Humor: 4/10 — pontual, nunca forçado
  • •Voz: "eu" — és o operador, não te escondes atrás de "nós"

📊 Vocabulário

Permitido: LLM, RAG, embeddings, agente, workflow, automação, MCP, fine-tune (se cliente é técnico).

Proibido: "leverage", "transformação digital integral", "ecossistema holístico", "sinergias", "paradigma", "disrupção", "unicórnio".

ICP típico

Sector: B2B PME-meio (20-200 funcionários)
Decisor: CTO, COO, Head of Operations
Sintoma: "perdemos X horas/semana em processo manual Y"
Não-cliente: Enterprise (procurement >6 meses)
              Microempresa (orçamento <5K)
              Quem quer "IA mágica" sem definir caso de uso

💡 Dica prática

Usa este template para clientes onde TU és a marca implementadora. Se tens equipa >2 ou MRR recorrente, vai para agencia-marketing em vez.

3

🏢 Template agencia-marketing

Para agências com 2-10 pessoas, MRR recorrente e 5-30 clientes em simultâneo. Diferença crítica face ao freelance: este template é da AGÊNCIA, não dos clientes dela.

🎯 Voice profile

  • •Formality: 5-6/10 — criativa, acessível
  • •Directness: 7/10 — clara mas com toque criativo
  • •Warmth: 7-8/10 — humano, com personalidade
  • •Humor: 5/10 — pode arriscar mais
  • •Voz: "nós" — és equipa, não pessoa única

⚠️ Não confundir voice da agência vs voice dos seus clientes

Se a agência tem 10 clientes, cria 10 sub-clientes dentro da estrutura (ou repos separados, se preferes). Cada um com a SUA voice. Este template só serve para LinkedIn/blog/email da agência em si.

Exemplo: agência widget-shop tem cliente acme. Voice de widget-shop ≠ voice de acme. Crias clients/acme/ separadamente.

Entregáveis típicos da agência

  • •Content calendar mensal por cliente
  • •Ads creatives (estática, vídeo, copy)
  • •Reports de performance mensais
  • •LinkedIn/blog/email da própria agência (para gerar pipeline)

💡 Quando NÃO usar agencia-marketing

Se és solo, mesmo com 5+ clientes, fica no freelance-ia. "Agência" implica equipa real (2+ humanos), não label vendido.

4

🎓 Template formador-online

Para criadores de cursos e/ou comunidade (ticket 200-2000€). A tua marca pessoal É o produto. Audiência nas redes sociais é canal principal de aquisição.

🎯 Voice profile

  • •Formality: 4-5/10 — didática mas próxima
  • •Warmth: 8-9/10 — caloroso, com personalidade vincada
  • •Vulnerability: 6-7/10 — admites incerteza, partilhas processo
  • •Frequency: alta — várias peças por dia

📊 Plataformas e registos

  • •LinkedIn: registo B (divulgativo) — posts, carrosséis, mini-essays
  • •X / Twitter: registo C (cercano) — threads, banter, takes rápidos
  • •YouTube: scripts adaptados ao formato (long-form vs shorts)
  • •Newsletter: B com toques de C — proximidade sem sobreposição
  • •Comunidade: C puro — mensagens informais, áudios

⚠️ Risco crítico: "academia genérica"

Se o output se sente como "qualquer formador online", a marca pessoal evaporou-se. Investe no voice profile com tics linguísticos próprios (palavras-assinatura, jeitos de começar, references culturais teus).

💡 Dica

Os 3 registos (A/B/C) são especialmente úteis aqui. Mesmo conteúdo, 3 versões automáticas: post LinkedIn (B), thread X (C), boletim (B+C). Reduz fricção de distribuição.

5

💼 Template consultoria-b2b

Para consultores com high-ticket (30-300K€) e engagements 3-12 meses em empresas médias/grandes. O cliente paga por seriedade, processo e autoridade.

🎯 Voice profile

  • •Formality: 8-9/10 — formal, autoridade
  • •Directness: 7/10 — clara mas sem brusquidão
  • •Warmth: 4-5/10 — profissional, não frio
  • •Humor: 1-2/10 — quase ausente
  • •Registo por defeito: A (formal)

✓ Entregáveis

  • ✓Propostas comerciais formais
  • ✓Decks (estado, recomendações, roadmap)
  • ✓Frameworks documentados
  • ✓Reports executivos mensais/trimestrais
  • ✓Briefings estruturados

✗ Evitar

  • ✗Memes, jokes, "uau"
  • ✗Emojis no entregável final (raríssimo)
  • ✗Casualismos ("tipo", "à la")
  • ✗"Eu acho" (em vez: "no nosso entender")
  • ✗Hot takes não fundamentados

Vocabulário-assinatura

"As evidências sugerem..."
"No nosso entender..."
"A análise das opções indica..."
"Recomendamos a implementação faseada de..."
"Os indicadores apontam para..."
"Considerando os trade-offs identificados..."

💡 Para LinkedIn pessoal do consultor

O consultor pode usar registo B em LinkedIn pessoal (thought leadership) e A nos entregáveis. Skills marketing-* detetam o canal e escolhem o registo certo automaticamente.

🛠️ add-client.sh: como criar cliente novo

Um comando, ~3 segundos, cliente pronto a trabalhar. O script add-client.sh valida o nome (kebab-case), clona um template, substitui placeholders e gera o CLAUDE.md de overrides.

Uso

bash scripts/add-client.sh <nome-cliente> [vertical]

# Exemplos
bash scripts/add-client.sh acme-corp freelance-ia
bash scripts/add-client.sh widget-shop agencia-marketing
bash scripts/add-client.sh north-star consultoria-b2b
bash scripts/add-client.sh nova-coach formador-online
bash scripts/add-client.sh xpto-startup vacio   # sem template

📊 O que o script faz (por baixo)

  1. Valida que o nome é kebab-case puro (regex ^[a-z0-9-]+$) — recusa "Acme Corp"
  2. Verifica que clients/<nome>/ NÃO existe (não sobrescreve)
  3. Cria estrutura: brand-context/{voice,positioning,icp,assets}, context/, projects/briefs/
  4. Clona ficheiros do template escolhido (voice-profile, positioning, icp, soul, user)
  5. Faz sed de {{CLIENT_NAME}} → nome real em todos os .md
  6. Gera CLAUDE.md do cliente com header, regras e referências ao brand-context
  7. Mostra a árvore criada e o próximo passo: cd clients/<nome> && claude

Estrutura resultante

clients/acme-corp/
├── CLAUDE.md                       # Overrides específicos
├── brand-context/
│   ├── voice/voice-profile.md      # Com placeholders do template
│   ├── positioning/positioning.md
│   ├── icp/icp.md
│   └── assets/
├── context/
│   ├── soul.md                     # Personalidade ajustada ao vertical
│   └── user.md                     # Template com campos típicos
└── projects/
    └── briefs/

💡 Placeholders

Mesmo com template, os ficheiros vêm com placeholders {{...}} que tens de preencher (cliente real, site, tom específico). O script avisa-te no fim com a lista exata. Sem preencher, skills marketing-* recusam-se a gerar copy.
