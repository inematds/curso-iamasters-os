📝 marketing-copywriting · a skill estrela

É o output mais frequente do OS no dia-a-dia. Gera copy para uma plataforma específica usando brand-voice e register A/B/C auto-detetado. Devolve 1-3 variações para tu escolheres. Passa pelo verifier obrigatoriamente.

Inputs requeridos

InputExemploSe faltar...
Tema/brief"Lançamento do curso v2"Pergunta
PlataformaLinkedIn / X / email / landingPergunta (muda tudo)
Purposeawareness / lead-gen / ofertaPergunta (muda tom)

🎚️ Register auto-detection

A skill mapeia plataforma → register automaticamente:

  • →Email premium / proposta = Registo A (formal)
  • →LinkedIn, blog, X thread, newsletter = Registo B (divulgativo, 95% do conteúdo)
  • →WhatsApp, DM, comentários, Stories = Registo C (próximo)

Podes forçar manualmente: "escreve em registo C". Útil quando o default está mal-calibrado.

💡 Dica

Pede sempre 2-3 variações em vez de "a melhor". Custa o mesmo em tokens e dá-te óptica para escolher. A "melhor" muda em função do contexto que só tu tens (cliente específico, evento da semana, mood).

♻️ marketing-content-repurposing

Converte 1 peça fonte em 5-8 peças multiplataforma respeitando brand voice. Uma talk de 60 min vira uma semana de distribuição em 15-20 min de trabalho teu.

Fontes suportadas (v0.2)

Vídeo YouTube (URL)

Invoca tool-youtube-transcript ou pede transcript manual

Transcript colado

Modo direto, mais rápido

Blog post (URL ou colado)

Usa tool-firecrawl-scraper

Reunião / call

Transcript Zoom/Fathom/Otter

Áudio local (MP3/WAV) não suportado v0.2 — pede transcript manual.

Pacote de output típico

→1 LinkedIn post (300-800 palavras)
→1 X thread (5-9 tweets)
→1 email newsletter (400-700 palavras)
→1 IG caption + sugestão visual
→3-5 clips ideas com timestamps
→3-5 headlines blog alternativos

🔗 Como compõe internamente

Para cada peça do pacote, invoca marketing-copywriting com a plataforma certa. Cada um passa pelo tool-output-verifier. Resultado: 5-8 outputs cada um passou pelo gate, não só o agregado.

3

📧 marketing-email-sequence

Cria sequências multi-email automatizadas: welcome, nurture, win-back, post-purchase, event-based, educational. Define gatilhos, cadência, segmentação e métricas-alvo.

TipoCadência típicaMétrica-alvo
Welcome / onboarding5-7 emails em 14 diasOpen >40%, ativação
Lead nurture1 email/semana, indef.CTR >3%, reply rate
Re-engagement3 emails em 10 diasRecuperar dormentes
Post-purchase4-6 emails em 30 diasOnboarding + upsell
Event-based3-4 antes + 1-2 depoisAtendimento + replay
Educational drip1 email a cada 2-3 diasCompletion rate

Avaliação inicial · o que pergunta

  1. Tipo de sequência (welcome / nurture / re-eng / post-purchase / event / educational)
  2. Audiência (segment, awareness stage, contexto da inscrição)
  3. Objetivo conversão (compra, demo, reply, completar onboarding)
  4. Tom + register (lê brand-voice se existe)
  5. Ferramenta de envio (Mailchimp, ConvertKit, n8n, ActiveCampaign)
  6. Restrições (legais, brand, frequência máxima)

⚠️ Não confundir com cold email

marketing-email-sequence é para fluxos automatizados com lista própria (alguém deu o email, está em opt-in). Para cold outreach (B2B prospecting), há a skill futura cold-email com regras diferentes (compliance, deliverability, sequência manual).

4

📐 marketing-positioning · deep dive

Vista superficial em Trilha 1.3. Aqui aprendes a executar bem: que perguntas fazer ao operador, como ler concorrentes, como apresentar ângulos para o operador escolher.

Process em 5 passos

  1. Recolher inputs — modo construção (sem positioning.md) ou refinamento (existe e queres iterar). 4-5 perguntas chave: o que vendes, para quem, alternativas atuais, ângulo emocional, alergias.
  2. Analisar concorrentes — pede 3-5 URLs de concorrentes, invoca tool-firecrawl-scraper, extrai positioning statements + palavras dominantes.
  3. Identificar gaps — onde todos dizem o mesmo, onde ninguém entra, qual a posição que ainda não tem dono no mercado.
  4. Propor 3-5 ângulos — vertical-specialist · use-case-specialist · método-próprio · anti-vende-fumos · stage-specialist (early/scale/turnaround). Cada um com prós/contras e exemplos.
  5. Operador escolhe — escreve para brand-context/positioning/positioning.md + propõe registar em decisions-log.

Quando invocar

  • Após marketing-brand-voice no onboarding
  • "Não diferencio bem"
  • "Todos dizem o mesmo que eu"
  • Mudança de mercado ou produto novo

Sinais de positioning fraco

  • Copy "consultor genérico"
  • Leads pedem desconto sempre
  • Concorrência sentida em todos os setores
  • Dificuldade em dizer "isto NÃO faço"
5

🎯 marketing-icp · deep dive

Sem ICP claro, copy e ads não segmentam. Esta skill combina interview com análise de clientes reais (3-5 bons + 1-3 maus) e extrai frases de validação que vão para os outputs.

As 6 perguntas canónicas

  1. Tens 3-5 clientes atuais ou passados que sejam "bons"? (pagaram sem regatear, recomendaram, repetiram) Nome, setor, dimensão, porquê.
  2. Tens 1-3 clientes "maus"? (queixosos, churn em 30 dias, sugadores de tempo) Nome, setor, porquê falharam.
  3. O que tinham em comum os bons antes de te contratar? (stage, sintoma, tentativas anteriores)
  4. Que palavras / frases eles usam quando descrevem a dor? (literal, não a tua versão polida)
  5. Qual o "trigger event" que normalmente faz contactar-te? (mudança de equipa, problema concreto, evento sazonal)
  6. O que NÃO queres como cliente? (anti-ICP — tipo de empresa, comportamento, expectativa)

📋 Output em brand-context/icp/icp.md

  • •Perfil ICP primário: setor, dimensão, role, stage, dor exata em 1 frase
  • •ICP secundário: se aplicável (não obrigatório)
  • •Linguagem: 10-20 frases literais que eles usam (para copy)
  • •Triggers: 3-5 eventos que normalmente precedem contacto
  • •Anti-ICP: 3-5 sinais que indicam "não para mim"
  • •Customer journey: stages awareness → consideration → decision (com objections típicas)

🔁 Refrescar regularmente

Reservar 30 min por trimestre para revisitar o ICP. À medida que vais ganhando clientes, o perfil "real" diverge do "ideal teórico". O ficheiro é vivo — não definitivo.

🔗 Padrão de skill collaboration

As skills não vivem isoladas. Compõem-se em pipelines. Saber o pipeline canónico ajuda-te a debugar quando uma skill "falha" — geralmente é um problema de composição, não de skill individual.

Pipeline canónica para qualquer output marketing

┌─────────────────────────┐
│  marketing-copywriting  │  ← gera 1-3 variações
└────────────┬────────────┘
             │
             ▼
┌─────────────────────────┐
│  tool-output-verifier   │  ← 4 checks (humanizer, voice, length, factuality)
└────────────┬────────────┘
             │
   score >=8 │ │ score <8
             │ │
             │ ▼
             │ ┌────────────────┐
             │ │ tool-humanizer │  ← reescreve com sugestões
             │ └───────┬────────┘
             │         │
             └────┐    │
                  ▼    ▼
            ┌──────────────┐
            │   Entrega    │
            └──────────────┘

📦 Contexto JSON passado entre skills

{
  "text": "...",
  "platform": "linkedin",
  "purpose": "awareness",
  "register": "B",
  "source_skill": "marketing-copywriting",
  "voice_loaded": true
}

Cada skill no pipeline recebe o JSON e pode adicionar campos (ex: verifier adiciona scores: {humanizer: 8, voice: 9}). Permite debugar: lê o JSON final e vês onde se partiu a cadeia.

⚡ Quando desativar gates

Para outputs internos (notas pessoais, brainstorming, drafts não-publicáveis), o verifier é overhead. Podes pedir explicitamente: "escreve sem gate, é interno". A skill marketing-copywriting respeita e salta o verifier nesse turno.

🚧 Falhas típicas no pipeline

  • Voice não carregada → verifier dá voice_score baixo → loop infinito. Causa: brand-context/voice/voice-profile.md vazio.
  • Length errado por platform → output X com 2000 caracteres em campo de 280. Causa: platform não passada no JSON.
  • Humanizer impossível → score nunca chega a 8 mesmo após 3 iterações. Causa: tema demasiado técnico ou voice mal definida.
