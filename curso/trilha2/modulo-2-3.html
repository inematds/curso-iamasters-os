🤖 tool-humanizer · anti-AI patterns

Deteta padrões de escrita IA num texto e devolve score 0-10 (10 = humano puro) + reescrita com sugestões. É a defesa final contra outputs "suspeitosamente IA".

AI-tell patterns que deteta

Vocabulário-sinal

delve, leverage, in conclusion, navigate, unlock, embark, tapestry, realm

Frases-template

"Let me know if...", "I hope this helps", "It's important to note that"

Estrutura

Bullet point com emoji a abrir cada um, headers + sub-bullets simétricos

Hedging excessivo

"may potentially", "could possibly", excesso de "I think"

📊 Modos de uso

  • Standalone: colas texto, dizes "humaniza isto" ou "tira o tom AI". Devolve score + reescrita comentada.
  • Pipeline: invocada por outra skill com JSON {text, context}. Devolve JSON estruturado para gate decision.
  • Dentro do verifier: tool-output-verifier inclui-a como um dos seus 4 checks.

🎯 Threshold típico

Bloquear se score <7. Score 7-8 passa mas com aviso (vê sugestões). Score >=9 é "humano-passable" sem reservas. Calibração varia por plataforma: WhatsApp tolera score 6, post LinkedIn público exige >=8.

2

✅ tool-output-verifier · gate 4 checks

O último guardião antes de entregar. Todas as skills marketing-* o invocam no último passo. 4 dimensões ortogonais, score 0-10 final, pass/fail.

1

Humanizer score

Delega em tool-humanizer. Score 0-10 sobre padrões AI. Peso típico no agregado: 30%.

2

Brand voice match

Lê brand-context/voice/voice-profile.md + register esperado. Compara vocabulário, ritmo, formalidade. Peso: 30%.

3

Length per platform

LinkedIn 300-800w · X 280 chars · email 150-700w · landing section 50-200w. Fora do range → flag. Peso: 20%.

4

Factuality flags

Deteta números, datas, nomes próprios, claims auditáveis. Não verifica — sinaliza para tu confirmares. Peso: 20%.

Exemplo de output JSON

{
  "verdict": "pass-with-warnings",
  "score": 7.6,
  "checks": {
    "humanizer": 8,
    "voice_match": 7,
    "length": 9,
    "factuality_flags": ["3 claims auditáveis", "1 número não-cited"]
  },
  "suggestions": [
    "Voice: muito uso de 'importante notar' — não é do teu registo B",
    "Confirmar: 'crescemos 40% YoY' (de onde sai este número?)"
  ]
}

🚪 Pass / fail criteria

  • pass: score >=8 e zero flags críticas
  • pass-with-warnings: score 6-7.99 ou flags não-críticas (entrega mas avisa)
  • fail: score <6 ou flag crítica (factuality em número específico) → volta para refinamento

🔥 tool-firecrawl-scraper

Wrapper de Firecrawl API para scraping sem bot blockers. Lê FIRECRAWL_API_KEY do .env; se faltar, degrada para WebFetch nativo e avisa. É a fonte de "dados frescos" do OS.

Process em 3 passos

  1. Verificar API key — lê .env. Se existe, usa https://api.firecrawl.dev/v1/scrape. Se não, degrada para WebFetch built-in e avisa o operador.
  2. Validar input — URL válido, scheme http/https, sem auth requerido. Rejeita ficheiros locais.
  3. Extrair markdown — Firecrawl devolve markdown limpo (sem ads, sem nav, sem footer). Cache local de 24h para evitar reqs duplicados.

Skills que dependem dele

  • →marketing-brand-voice — extrai voice samples de web/LinkedIn
  • →marketing-positioning — analisa concorrentes
  • →marketing-content-repurposing — lê blog posts longos
  • →welcome-quick-win — extrai conteúdo do URL público do operador
  • →strategy-web-research — múltiplos URLs em paralelo

💸 500 créditos grátis

Regista-te em firecrawl.dev (500 créditos grátis, ~500 páginas). Adiciona FIRECRAWL_API_KEY=fc-xxx em .env do repo. Para uso pesado: $19/mês cobre 3.000 páginas.

⚠️ Degradação graceful

Se a API key falta, a skill não bloqueia — tenta WebFetch nativo. Mas avisa: "Sem Firecrawl API key. Algumas webs com bot-blockers (LinkedIn, sites com Cloudflare agressivo) vão falhar. Considera ativar."

4

🎨 tool-visual-explainer · HTML partilhável

Gera HTML5 autocontido (1 ficheiro, sem dependências externas, mobile-first) para partilhar análises, comparativos, recaps. Paleta sóbria com acento laranja iAmasters. Inspirado na skill visual-explainer da suite Anthropic.

✓ Quando usar

  • Outras skills fecham com análise para partilhar
  • "Faz-me HTML disto", "põe bonito"
  • Apresentar a cliente, sócio, conselheiro
  • Tabelas longas (4+ linhas, 3+ colunas)
  • Quick-win, six-hats output, positioning

✗ Quando NÃO usar

  • Outputs internos (só o Claude lê)
  • Já estás em ferramenta visual (Notion, Figma)
  • Output <200 palavras (markdown chega)
  • Conteúdo dinâmico que muda diariamente
  • Quando o cliente quer texto plano para copiar

Características técnicas

  • 1 ficheiro HTML com CSS + JS inline (sem CDN exceto Tailwind opcional)
  • Mobile-first — funciona em ecrã 360px
  • Paleta: dark mode default (slate 900), acento laranja iAmasters (#FACC15), toggle light/dark
  • Tipografia: Inter via Google Fonts (única dependência externa)
  • Tamanho típico: 30-80 KB. Cabe em DM, WhatsApp, email attach.
  • Acessibilidade: contraste AA mínimo, headers hierárquicos, labels em forms

📤 Output path

Salva em projects/visual/YYYY-MM-DD-titulo-curto/index.html. O operador depois copia-o para onde quiser (Drive, Notion attach, S3, GitHub Pages).

🚫 Quando NÃO usar uma skill

Skills têm custo: tokens (lêem ficheiros de contexto), tempo (passos extra como verifier), e ruído (mais ativações falsas). Aplicá-las a tudo é como pôr 5 cintos para descer à esquina.

CasoSkill?Porquê
"Que dia é hoje?"NãoPergunta factual, prompt direto
Resumo de 1 parágrafo só para tiNãoOutput interno, sem leitor externo
Brainstorm exploratório livreNãoSkills constrangem; queres soltar
Email pessoal a amigoNãoBrand voice é overhead aqui
Post LinkedIn públicoSimcopywriting + verifier obrigatório
Análise de decisão grandeSimsix-hats + decisions-log
Entregável para clienteSimPipeline completa com gates

🎚️ Override manual

Mesmo que a skill ative automaticamente, podes desativar para um turno: "escreve sem skill, é interno" · "prompt direto, sem voice carregada" · "salta o verifier". O agente respeita.

📢 Ruído de descriptions

Quanto mais skills tens instaladas, mais descriptions o Claude lê no início de cada sessão. Cada description "boa demais" pode ativar em casos não intencionais. Regra: se uma skill ativa <1x por mês, considera desinstalá-la (mover para _optional/ ou remover do registry).

📦 Camada 2 e skills opcionais

A Camada 1 são as 23 skills core. A Camada 2 é uma biblioteca on-demand instalável via /install-skill <github-url>. Para expandires o OS sem inchá-lo.

Como funciona /install-skill

  1. Clona localmente para /tmp/skill-validation/ (não toca em .claude/skills/ ainda)
  2. Valida estrutura — frontmatter completo, description 50-500 chars, sem secrets em commits
  3. Mostra preview ao operador — o que faz, quando ativa, dependências
  4. Pede confirmação — "Instalar em .claude/skills/<categoria>/<nome>? [y/N]"
  5. Atualiza catalog + CLAUDE.md registry + invoca wrap-up para commit

🏪 Skills opcionais conhecidas

cognito (Luis Pitik)

Sistema operativo de pensamento. /install-skill cognito.

docx · xlsx · pdf · pptx (Anthropic)

Gera documentos Office reais. /plugin install anthropic-skills.

silver-platter, frontend-design, website-intelligence

Skills de produção alta de outras suites. Ver docs/skills-recommended.md.

🛡️ Anti-inflação

A validação local antes de globalizar evita o cenário "instalei 80 skills do GitHub, agora o Claude ativa as erradas em metade dos turnos". Cada skill nova é uma decisão consciente, não um clone reflexo.
